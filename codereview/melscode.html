There are 3 main functions, one iterates through the data array (full of ojectst), one submits your answer and changes the money according to your answer, and one that seems to create the modal. I would suggest breaking the functions up into smaller, more specific functions and then maybe nesting some of the functions within each other. Some of the functions do multiple things and was a bit hard to follow all of it. Also a smaller tidbit, would to use an API (or database?, im still unsure lmao) for the questions without having to hardcode everything, but its essentially the same result anyway.
